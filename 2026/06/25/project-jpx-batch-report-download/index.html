JPX 报告批量下载工具 — 日本上市公司资料整理脚本（RA）

项目背景

这是我在 2026 年某华五高校担任研究助理（RA）期间第一周的工作产出。任务核心是从日本 JPX / EDINET 平台批量下载上市公司的年报和可持续发展报告，按公司编号自动归档，并把下载结果写回 Excel 检查表。

项目代码完全开源：可以下载所有脚本直接运行。目标是让后来接手这类任务的同学拿到后照着操作文档也能跑起来，不至于被繁琐的资料整理消磨学术热情。

解决的问题

研究项目中，经常需要批量获取日本上市公司的公开报告。手动逐家下载——打开每个公司的 EDINET 页面、查可持续发展网页、下载 PDF、改文件名、记录完成情况——效率极低且容易出错。

这个项目将整个流程自动化，三件事情一步到位：

  1. 下载每家公司的年报，统一放入 report 文件夹
  2. 下载可持续发展相关报告（ESG / CSR / 环境报告），统一放入 sreport 文件夹
  3. 把下载结果写回 list.xlsx 检查表

三个核心脚本

脚本 功能
download_reports.py 主下载脚本，负责年报 + JPX 直接挂出的可持续 PDF，同时更新 Excel
supplement_sustainability_webpages.py 当公司没有可持续 PDF 时，从 JPX 公司治理报告中定位网页地址
mine_sustainability_pdfs.py 进一步挖掘公司官网的 sustainability / CSR / ESG 页面中隐藏的 PDF

三个脚本都支持重复运行，已下载的文件会复用，不会每次从头来。

准备工作

使用前需要配好 EDINET API Key（在 EDINET 官网注册邮箱并生成密钥），然后在项目目录创建 .env 文件写入密钥即可。

结果分类

每条记录最终会归入以下几种情况：

  • 有年报 + 有可持续 PDF：最完整的结果，report 和 sreport 都有文件
  • 只有网页没有 PDF：F 列标记为「无报告，有网页内容」，G 列记录网页地址
  • 什么都没找到：标记为「无可持续声明」
