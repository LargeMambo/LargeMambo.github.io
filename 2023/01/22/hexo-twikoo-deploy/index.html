为什么选 Twikoo

GitHub Pages 托管的静态博客没有服务端，无法直接处理评论。主流的免费方案有三种：

方案 国内可用 访客门槛 部署复杂度
Twikoo + Vercel + MongoDB ✅ 无需登录 低
Waline + Vercel + LeanCloud ✅ 无需登录 低
Giscus（GitHub Discussions） ❌ 被墙 需 GitHub 账号 极低

Twikoo 国内访问稳定，Butterfly 原生支持，配置简单，是目前中文博客圈最常用的方案。

整个部署只需要三个服务，全部免费：

  • MongoDB Atlas：免费云数据库，存储评论数据
  • Vercel：免费云函数平台，运行 Twikoo 服务端
  • 你的博客：前端评论框

第一阶段：创建 MongoDB 数据库

1.1 注册并创建集群

  1. 打开 mongodb.com/cloud/atlas，注册免费账号
  2. 登录后点击 Create a deployment
  3. 选择 M0（免费套餐，512MB 存储，评论数据足够用几十年）
  4. 云服务商选 AWS，地区选 Singapore（国内延迟最低）
  5. 集群名保持默认 Cluster0，点 Create Deployment

1.2 创建数据库用户

  1. 左侧菜单 → Security → Database Access → Add New Database User
  2. Authentication Method 选 Password
  3. 填写用户名和密码，记下来备用
  4. 下方 Database User Privileges 选 Read and write to any database
  5. 点 Add User

1.3 放行所有 IP

Vercel 的函数节点 IP 是动态变化的，需要允许所有 IP 访问数据库：

  1. 左侧菜单 → Security → Network Access → Add IP Address
  2. 点 Allow Access from Anywhere，地址自动填入 0.0.0.0/0
  3. 点 Confirm

1.4 获取连接字符串

第二阶段：部署 Twikoo 到 Vercel

2.1 一键部署

  1. 打开 vercel.com，用 GitHub 账号登录
  2. 在浏览器中打开以下地址，触发一键部署：
  1. 页面会要求在 GitHub 创建一个仓库来存放 Twikoo 的服务端代码：
    • Git Scope：选你的 GitHub 账号
    • Private Repository Name：填 twikoo（或任意名字）
  2. 点 Create → 再点 Deploy，等待部署完成

2.2 添加环境变量

部署完成后，Vercel 会跳到项目概览页：

  1. 点顶部的 Settings 标签
  2. 左侧菜单找到 Environment Variables
  3. 添加一条变量：
    • Key：MONGODB_URI
    • Value：第一阶段修改好的 MongoDB 连接字符串
  4. 确认右侧三个环境（Production / Preview / Development）全部勾选
  5. 点 Save

环境变量保存后不会自动生效，必须重新部署才能读取到。

2.3 重新部署

  1. 点顶部 Deployments 标签
  2. 找到最新那条记录，点右侧 ··· 三个点
  3. 点 Redeploy → 弹窗中再次点 Redeploy
  4. 等状态变为绿色 Ready

  • 连接字符串中是否有 /twikoo 数据库名

第四阶段：配置管理面板

4.1 登录管理面板

Twikoo 的管理后台不在 Vercel 地址，而是藏在博客评论区里：

  1. 打开博客任意文章，滚到底部评论区
  2. 找评论框右下角的齿轮图标 ⚙️，点击
  3. 首次进入会提示设置管理员密码，设好后登录

4.2 设置必填字段

4.3 开启邮件通知（可选）

有人回复评论时，自动向访客发送邮件提醒，需要配置 SMTP：

在管理面板通用配置中填写以下几项（以 Gmail 为例）：

配置项 说明 示例
SMTP_SERVICE 邮件服务商 Gmail
SMTP_HOST SMTP 服务器地址 smtp.gmail.com
SMTP_PORT 端口 465
SMTP_USER 发件邮箱 yourname@gmail.com
SMTP_PASS 邮箱授权码（不是登录密码） xxxxxxxxxxxx
SENDER_EMAIL 显示的发件人邮箱 yourname@gmail.com
SENDER_NICK 显示的发件人昵称 博客通知

国内常用邮箱参考：

  • QQ 邮箱：SMTP_SERVICE = QQ，端口 465，授权码在邮箱设置中生成
  • 163 邮箱：SMTP_SERVICE = 163，端口 465，授权码同理

日常维护

查看和管理评论： 博客文章底部 ⚙️ → 登录 → 评论管理，可以删除垃圾评论。

Twikoo 版本更新： 进入 Vercel 项目 → Deployments → Redeploy，会自动拉取最新版本。

数据备份： MongoDB Atlas 控制台 → Clusters → Browse Collections，可以导出评论数据。
